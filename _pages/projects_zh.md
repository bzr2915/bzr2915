人造出工具，工具反过来重塑人——工程史上，这般角色互换不止发生过一次。草原上的游牧者驯服野马，本意是借马力走得更远；骑术成熟之后，骑手已不再是走得快一些的步行者——骑兵开疆千里，马背催生了全新的力量形态。蒸汽机带来的逆转更为彻底：肌肉曾是劳动的主要原动力，蒸汽机出现之后，工匠不再出力，而是管理力的释放。**工具接管执行，人决定做什么。**

*芯片设计六十余年的演进，正是又一次这样的逆转——而且，或许是迄今最深刻的一次。* 起初，设计者独坐绘图桌前，铅笔、直尺、一颗能同时记住上百个约束的大脑，便是全部的"设计系统"。后来软件工具兴起，设计者从亲手画线的匠人升格为驾驭工具的指挥者。再后来机器学习进入设计流程，工具获得了"眼力"——能预测、能搜索、能在人看不到的角落发现可能性——但拿起工具的手，仍然是人的手。而今天，新的转折已经到来：**AI不再是被人调用的助手，而正在成为自行构思策略、选择工具、评判结果的设计主体。** 人没有消失，但主动权的重心确确实实地转移了。我们以四幕叙事讲述这段历程。

## 手工作坊：人是唯一主体

{% include figure.html path="assets/img/research/act1_artisan.png" class="img-fluid rounded z-depth-1" alt="手工作坊时代——工程师在绘图桌前独自完成电路设计" %}

1958年秋天，杰克·基尔比在德州仪器达拉斯实验室的工作台上，做出了人类第一块集成电路。一小片锗晶体，上面连着晶体管、电阻和电容，以细如发丝的金线相接。手边的工具跟一百年前的制图匠人别无两样：铅笔、直尺、坐标纸、一双稳健的手。导线走向、结点位置、寄生效应，都先在脑中成形，再一笔一划誊写到纸面上。验证呢？也只能靠记忆——记住设计意图，逐一核对有没有画错。**那个年代，设计者的大脑就是唯一的设计工具。** 能设计多复杂的东西，上限就是人脑能想多复杂的事。

初时尚可应付。但1965年，戈登·摩尔在仙童半导体观察到一条日后被称为"摩尔定律"的规律：芯片上的元件数目大约每两年翻一番。物理复杂度沿指数曲线攀升，认知能力却不会跟着指数攀升。工作记忆有限，注意力本质上串行，人的大脑不会因为又过了一年就多出一倍容量。到1970年代初，电路的规模已膨胀到任何个人都无法在脑中从头到尾走完一遍设计。问题不在材料，不在制造——**问题在认知**：硅能容纳的东西越来越多，而人能同时想清楚的东西并不会跟着变多。出路不在于培养更聪明的人，而在于发明全新的工具。

## 工具时代：人驱动工具

{% include figure.html path="assets/img/research/act2_tools.png" class="img-fluid rounded z-depth-1" alt="工具时代——工程师如指挥家般驾驭EDA软件工具" %}

1973年，伯克利的拉里·内格尔在唐纳德·彼得森指导下写出了SPICE程序——能在计算机里模拟电路行为的仿真器。此前，要知道电路能不能用，最靠谱的办法就是造出来量一量。有了SPICE，工程师第一次可以在制造之前，用数学审问设计。**先仿真，后制造**——今天听来理所当然，在当时却是认知上的解放。

此后是一连串自动化的胜利。新思科技的Design Compiler能把硬件描述语言写成的逻辑自动综合为门级网表；伯克利的TimberWolf系统用模拟退火算法完成布局布线，质量有时甚至超过最好的人工。算法逐一接管了手工时代的体力劳作，学科也有了与新面貌相称的名字：**电子设计自动化**（EDA）。

由此确立的范式，清晰而稳固，一用就是三十年。从架构选型到约束设定，从策略选择到结果判定，全由人拍板。工具是谓语：忠实、迅捷、规模远超人手——但**不会提议，不会质疑，不会自作主张**。SPICE不会建议换拓扑，布局引擎不会争辩该用另一种平面规划。百分之九十九的体力劳作已经交给了软件，但最后百分之一——需要经验、直觉和判断力的战略决策——依然牢牢攥在人的手里。工具再强大，说到底还是锤子；而**锤子从不过问该造什么**。

格局缔造了奇迹：数十亿晶体管的微处理器，基尔比若在世恐怕会视为外星文明之作的片上系统。但底层范式始终未变。每一代EDA工具，都仍然需要懂得配置参数、解读输出、承担最终责任的人站在背后。工程师从制图员变成了指挥，但指挥到底还站在台上，握着指挥棒，对每个音符负责。当芯片规模从百万晶体管攀至十亿、百亿，设计规则从薄薄一本膨胀为令人窒息的巨册，不安浮现了：当连指挥者本人的技艺——配置、约束、判断——也复杂到超出心智承载力时，接下来该怎么办？

## AI辅助：给锤子装上眼睛

{% include figure.html path="assets/img/research/act3_ai_assisted.png" class="img-fluid rounded z-depth-1" alt="AI辅助时代——装上AI之眼的锤子，但仍被人类之手紧握" %}

2010年代给出的回答，名字新鲜，野心却古老：机器学习。神经网络无需跑完综合流程便能预判时序收敛与否；贝叶斯优化用远少于穷举的仿真次数即可调谐参数；强化学习智能体则在布局问题中自行探索摆放方案。乍看像是根本性的变革——工具终于要从人手中夺过主导权了。但细看格局，变化并没有那么激进。设计流程仍是人定义的，在哪个环节用哪种模型仍是人挑选的，结果好不好仍由人来裁定。**AI被嵌入了现有的EDA流水线，充当的是加速器**——更聪明的零件装进了架构未变的机器。给锤子装上眼睛，瞄得准了许多，但拿起锤子、挥动锤子的，依然是人。

我们的研究故事，从模拟电路综合——芯片设计中最后的手工作坊——讲起。与早已高度自动化的数字综合不同，2010年代的模拟设计仍高度依赖老师傅的经验直觉：调晶体管宽度、设偏置电流、配补偿网络，门道需要多年磨练，且难以写成规则交给程序。核心是评估代价极高的黑盒问题：拟定一组器件参数，跑一次SPICE仿真——可能耗去几分钟甚至几小时——看结果达不达标，然后再试。空间浩大、维度极高、每一步都很贵。我们问的第一个问题很朴素：能不能让AI少花仿真代价，就把好的设计参数找到？

最初的回答是贝叶斯优化：为仿真器的输入输出关系建概率代理模型，用模型指引搜索方向，而不是漫无目的地遍历。早期效果不错，但维度很快成了问题——实际模拟电路可能有五十到两百个可调参数，高斯过程在如此高的维度下性能急剧下降。维度瓶颈推动了此后近十年的方法探索。我们找到了子空间分解的路径：在高维参数空间中辨识出仿真器响应最敏感的低维流形，把建模精力集中在那里，由此发展出**tSS-BO**和**BBGP-sDFO**一族方法。子空间不够用时，转向域分解——用Voronoi树把搜索空间切成分区，在各分区并行运行局部优化器，即**cVTS**和后来精炼的D3PBO。多目标的挑战——功耗、速度、面积、噪声要同时兼顾——催生了帕累托搜索策略VTSMOC以及层级多目标框架HiMOSS。同期，实验室另一条线索走了截然不同的路：强化学习。不把电路调参当作一次性优化来解，而是看作序贯决策过程——智能体像老师傅一样，调一下偏置电流，看看带宽变了没有，再决定下一步怎么动。多智能体强化学习框架**EVDMARL**让我们能把复杂电路拆成若干功能子模块，每个子模块交给专门的智能体管理，如同设计团队按既定接口分工协作。

为模拟电路打造的方法，后来意外地展现出跨领域的通用性。我们把同样的核心思路——代理建模、智能搜索、分解策略、多目标推理——带到了微处理器设计空间探索（**APPLE**、**ROI-HIT**）、物理仿真加速（热分析求解器**pPIRW**、电源网络IR降预测方法**CRC**），以及开放基准建设（**AnalogGym**——缺少可复现的比较，所谓进步不过是在暗处摸索）。从模拟调参的朴素问题出发，十年间自然生长为一整套AI辅助EDA的方法体系。体系的统一性不在于技术手段的雷同，而在于背后始终如一的核心追问：怎样用学习的方法，让昂贵的黑盒设计问题变得可解？

然而算法能以十年前不可想象的效率导航高维空间，能平衡多个相互竞争的目标，能把放大器上积累的经验迁移到下一个放大器——成果扎实，进展显著。但**设计流程的底层架构并没有变**。AI再精巧，仍然是人在预设流程的既定阶段上插入的模块。人仍然决定"解什么问题""用什么工具""什么叫足够好""什么时候收手"。那么，**假如拿起锤子的，不再是人的手呢？**

## AI原生：主客体翻转

{% include figure.html path="assets/img/research/act4_ai_native.png" class="img-fluid rounded z-depth-1" alt="AI原生时代——AI大脑在指挥中枢调度设计工具，人类在上方监督" %}


"AI原生EDA"不是"往EDA里加更多AI"——不是给旧脚手架刷一层更厚的智能漆。而是设计的语法结构被改写了。过去，人是主语，AI是偶尔被调用的宾语——快些的优化器，聪明些的预测器，镶嵌在人所编排的流程里，受人的最终裁决约束。现在，**AI成了主语**。AI自行构想设计策略，像匠人打量工作台上的家伙什一样审视可用的EDA工具，挑选、调用、解读结果、自主决定下一步。工具没有消失，而是成了智能体的设计语汇——AI作用于硅的一组动词。人也没有消失，只是转为监督者的角色——不再执行，而是界定什么问题值得解。**过去是人用AI增强的工具来设计，现在是AI自行调用工具完成设计——主客体翻转了。**

沿着这个方向，我们首先提出了**MARIO**——元优化框架。出发点很简单：如果AI能把优化算法用好，就应该能自行判断该用哪种。MARIO不固守单一策略，而是同时维护一组算法——进化的、代理模型的、无梯度的、混合的——实时监测各自进展，动态把算力往表现好的策略上调，从停滞的策略中撤回。换言之，AI在优化"优化本身"。

在MARIO基础上，我们提出了**ATOM**。此前问的是"给定电路架构，最好的器件尺寸是多少"；ATOM问了更根本的问题——"**什么架构首先应该存在？**"晶体管、电容、电阻有无穷的连接方式，构成巨大的组合空间——ATOM在其中探索，提出人类设计者不易想到的拓扑方案，评估的速度和广度远超个人心智。MARIO让AI自主治理优化过程，ATOM更进一步：**在定尺度之前，先定形状。**

我们最新的工作是**Atelier**——基于大语言模型的多智能体系统。在Atelier中，电路设计的完整弧线，从规格书到验证通过的结果，由各司其职的AI智能体协作完成。有的分析需求、拆解子问题，有的搜索拓扑空间，有的修改和精炼候选架构，有的调谐参数，还有一个独立充当决策者——裁定取舍、接受或驳回方案、指引下一轮迭代。智能体之间沟通协商，偶尔产生分歧，如同经验丰富的设计团队——只不过运转在机器速度上，同时考量的方案远多于任何人类团队。**人给出规格与约束，其余交给AI。**

回头看这三个系统，共享的不是算法，而是一个渐进的趋势：AI在设计循环中承担的角色越来越主动——从选择策略（MARIO），到构想架构（ATOM），再到统筹全流程（Atelier）。人的角色并未丧失分量——界定对了问题的监督者，胜过一千个高效解决错误问题的执行者。但主动权——循环中"下一步做什么"由谁说了算——确实在安静而深远地转移。匠人先是成了工具制造者，又成了指挥，如今正向策展人——界定问题、评鉴成果的人——演变。**拿起锤子的，已不再是人的手。**

## 未完成的翻转

然而，果真如此吗？

前路隐入了尚不可辨的雾中。以超人效率优化指标的机器，与能判断"何为好设计"的心智之间，差距也许远超我们的想象。好设计不只是满足指标——还意味着架构中蕴含优雅，没有任何损失函数会去奖励的那种优雅；意味着预见两代工艺节点之后的制造风险；意味着凭尚无法形式化的直觉感知到：窗口就在此刻，过了就关上了。**优化是技术；判断是修养。** 两者之间的鸿沟能否被弥合——靠规模、靠架构、还是靠尚未被想到的东西——仍然是我们这一行悬而未决的根本问题。

也许该说：人的不可替代性并没有消失，只是如同此前每一次翻转那样，又向更高的抽象层迁移了。既非执行，也非编排，而是发问。**提出问题，比回答问题更根本。** 据我们所知，没有任何算法学会了那个先于一切设计的行为：判定哪些问题值得追问、哪些难题配得上文明有限的心力。
